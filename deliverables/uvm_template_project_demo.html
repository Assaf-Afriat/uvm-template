UVM verification template

A minimal registered-adder DUT with valid handshake, UVM agent (config-driven virtual dut_if), scoreboard, functional coverage, and bound SVA. Package pkg/dut_pkg.sv pulls in class files only; interface/dut_if.sv compiles separately. Same scripts/Run/run.py workflow as the ALU project (Questa built-in mtiUvm; set macro path to your install’s UVM src via UVM_SRC).

Design specification

Condensed from dut/dut.sv and interface/dut_if.sv. Placeholder RTL: registered adder — result = a + b, valid_out tracks valid_in.

valid_in + a, b ──► dut ──► result[8:0], valid_out

Interface signals

SignalWidthRole
clk, rst_n1Clock, active-low reset (passed around dut_if)
a, b8Operands (randomized in sequences)
valid_in1Stimulus valid
result9Sum output
valid_out1Result valid

Package vs interface

  • pkg/dut_pkg.sv includes UVM classes only — not dut_if.
  • compile.do runs vlog interface/dut_if.sv before tb_top.sv so virtual dut_if resolves.

Source of truth

  • RTL: dut/dut.sv
  • SVA: sva/dut_assertions.sv (bound in tb_top.sv)

Authoritative RTL: dut/dut.sv.

Quick run

Same entry point as the ALU project. Execute from scripts/Run/. Set Questa UVM_SRC (and macro path) to your install’s UVM source directory under verilog_src.

cd scripts/Run python run.py python run.py --test dut_base_test python run.py --gui python run.py --clean python run.py --help

Prerequisites: QuestaSim on PATH, Python 3.x. Default test is dut_base_test when wired in elaborate.do / run.py.
